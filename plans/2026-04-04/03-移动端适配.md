# 移动端适配计划（动态组件方案）

## Context

当前 day 视图的移动端体验差：
- 侧边栏（280px）与 Timeline 并排在 375px 屏幕上导致 Timeline 仅剩 ~95px 可用宽度
- 任务卡片无法正常显示内容
- 桌面端的 Timeline 时间轴布局在移动端本质上不适合

**目标**：采用动态组件方案，PC 和 Mobile 完全分离布局

---

## 方案对比

| 方案 | 原理 | 适用场景 |
|------|------|----------|
| **响应式（RWD）** | 同一套 HTML + CSS Media Queries | 布局相似，仅间距/字体差异 |
| **动态组件** | 检测设备，动态加载不同组件 | 布局差异大的场景（本项目） |
| **SSR 自适应** | 服务端检测 UA 返回不同 HTML | 首屏性能敏感 |

本计划采用**动态组件方案**

### 动态组件实现

```vue
<!-- index.vue -->
<script setup>
import { computed } from 'vue'
import { useMobile } from '@/composables/useMobile'
import { useDayNavigation } from './composables/useDayNavigation'
import { useDayModal } from './composables/useDayModal'
import { useDayData } from './composables/useDayData'
import { useDailyReport } from './composables/useDailyReport'
import { useNotifications } from '@/composables/useNotifications'

const { isMobile } = useMobile()

// 动态导入，代码分割
const DayView = computed(() => 
  isMobile.value 
    ? defineAsyncComponent(() => import('./MobileView.vue'))
    : defineAsyncComponent(() => import('./DesktopView.vue'))
)
</script>

<template>
  <Suspense>
    <DayView />
  </Suspense>
</template>
```

### Step 6: 重构 index.vue

**修改文件**: `src/views/day/index.vue`

简化，只保留动态组件逻辑和共享数据层

```vue
<script setup>
import { computed } from 'vue'
import { useMobile } from '@/composables/useMobile'
import { useDayNavigation } from './composables/useDayNavigation'
import { useDayModal } from './composables/useDayModal'
import { useDayData } from './composables/useDayData'
import { useDailyReport } from './composables/useDailyReport'
import { useNotifications } from '@/composables/useNotifications'

const { isMobile } = useMobile()
const { isReady, isLoading, scrollToTask, currentHour } = useDayNavigation()
const { showAddModal, editingTask, openAddModal, openEditModal } = useDayModal()
const { dailySchedule, carryOverUncompletedTasksTo, fetchTasks, setLoading } = useDayData()
const { reportVisible, reportStats, closeReport } = useDailyReport()
const { startListening, stopListening, clearNotifiedHistory, requestPermission } = useNotifications()

// 动态组件
const DayView = computed(() => 
  isMobile.value 
    ? defineAsyncComponent(() => import('./MobileView.vue'))
    : defineAsyncComponent(() => import('./DesktopView.vue'))
)

// 共享逻辑挂载到 window 供子组件访问（Vue 3 provide/inject 更优，此处示意）
</script>

<template>
  <div class="h-screen w-full bg-background flex overflow-hidden font-sans text-foreground relative selection:bg-primary selection:text-primary-foreground">
    <Suspense>
      <component :is="DayView" />
    </Suspense>
  </div>
</template>
```

## 注意事项

1. **Composables 共享**：所有 `useDay*` composables 逻辑不变，只需在 index.vue 统一调用后通过 `provide` 传递给子组件
2. **响应式检测**：`useMobile` 在组件 mount 时才生效，需确保 Suspense 加载后再渲染
3. **代码分割**：动态组件会自动代码分割，桌面端不会加载移动端组件代码
