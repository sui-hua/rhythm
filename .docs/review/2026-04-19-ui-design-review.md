# UI 设计审查员报告

本报告聚焦当前项目的视觉层级、一致性、控件可识别性、布局节奏和整体风格统一性。结论基于现有 Vue 组件、样式实现，以及仓库内截图 `screenshots/01-day.png`、`screenshots/02-habits.png`、`screenshots/03-direction.png`、`screenshots/04-month.png`。

## 问题 1：信息层级过度依赖超大数字，导致主任务与辅助信息失衡
- 标题：超大日期/数字抢走主内容注意力
- 证据（组件/页面/样式）：
  - `src/views/day/components/Sidebar.vue:22-38` 中日期标题使用 `text-4xl font-black italic uppercase`，而任务列表标题只有 `text-sm`
  - `src/views/month/components/DayCell.vue:23-29` 中日期数字提升到 `text-4xl sm:text-5xl lg:text-6xl`
  - 截图 `screenshots/04-month.png` 中，用户首先看到的是整屏粗黑数字，而不是任务密度或当前关注日
- 问题：
  - 页面把“日期数字”做成了绝对主视觉，但真正驱动操作的内容是任务、计划和状态。当前层级把装饰性信息放大，把操作性信息压缩，导致用户看见页面时知道“这是日期”，却不容易快速判断“今天要做什么”。
- 原因：
  - 设计语言明显偏海报化排版，但没有为业务信息建立第二层稳定层级，结果大数字成为唯一强视觉锚点。
- 影响：
  - 日视图侧边栏里任务标题和编辑入口被压到次要层，扫描效率变差。
  - 月视图更像静态年历海报，而不是可操作的任务总览，降低点击进入单日的欲望和准确性。
- 建议：
  - 降低日期数字 1 个层级，例如日侧边栏从 `text-4xl` 降到 `text-2xl`，月格子从 `text-6xl` 降到 `text-3xl~4xl`。
  - 为任务相关信息建立更稳定的二级层级：任务数、任务状态、今日标签、当前日高亮要比装饰性刻度更先被看到。
  - 月视图可给“今天”或“有任务的天”增加明确容器态高亮，而不是只靠边缘刻度和细小点阵。
- 优先级：高

## 问题 2：交互入口大量隐藏在 hover 态里，控件可识别性不足
- 标题：编辑、导航、面板操作依赖悬停才出现
- 证据（组件/页面/样式）：
  - `src/components/Navbar.vue:8-16, 87-95` 桌面导航默认完全隐藏，通过顶部热区 hover 才出现
  - `src/views/day/components/Sidebar.vue:79-85, 114-120` 任务编辑按钮 `opacity-0 group-hover:opacity-100`
  - `src/views/habits/components/HabitSidebar.vue:35-40, 62-67` 习惯编辑入口同样默认隐藏
  - `src/views/direction/components/DirectionSidebar.vue:31-33, 175-177, 195-196` 目标设置按钮只有 hover 后出现
- 问题：
  - 许多关键操作入口只有“鼠标已经悬停到正确位置”后才显露。视觉上会让列表项更干净，但牺牲了“可编辑”“可进入下一步”的提示。
- 原因：
  - 当前设计过分追求极简和留白，把交互提示也一起收掉了；尤其在信息密度本就不高的页面里，用户很难预判每个块能做什么。
- 影响：
  - 新用户不容易发现任务、目标、习惯都支持编辑。
  - 导航栏在桌面端处于“默认不存在”的状态，容易让页面显得像孤岛。
  - 对触控板、键盘用户和低熟练度用户不友好。
- 建议：
  - 保留 hover 强化，但不要把入口完全隐藏；可改为默认 40%~60% 可见，hover 再增强。
  - 桌面导航建议改为默认半透明可见，或至少保留一个固定的轻量导航锚点，而不是完全依赖顶部热区。
  - 列表行尾统一使用弱显图标或更多菜单，形成一致的“可编辑”暗示。
- 优先级：高

## 问题 3：浅灰系统用得过重，导致边界、状态和可点击性都偏弱
- 标题：对比度过低，组件边界和状态差异不够明确
- 证据（组件/页面/样式）：
  - `src/assets/tw-theme.css:56-65` 全局 `secondary`、`muted`、`border` 都非常接近浅灰白
  - `src/views/direction/components/MissionBoardMonthHeader.vue:133-150` 空标题和折叠按钮都压到 `text-muted-foreground/30`、`/40`
  - `src/views/month/components/DayCell.vue:18-20, 39-42` 任务点与刻度线大量使用 `bg-foreground/20`、`bg-zinc-100`、`opacity-30`
  - `src/views/login/index.vue:23-30, 36-43` 登录表单仍使用一套单独的浅灰边框输入框，与全站通用输入组件不一致
  - 截图 `screenshots/03-direction.png`、`screenshots/02-habits.png` 可见很多卡片、边框、标签彼此亮度差过小
- 问题：
  - 页面整体“很轻”，但轻到边界不明确、按钮不够像按钮、次级信息几乎接近禁用态。用户需要更长时间确认哪里是容器、哪里可点、哪里只是装饰。
- 原因：
  - 主题变量和页面局部类名都偏向极淡灰阶，且多处再叠加透明度，导致局部有效对比继续下降。
- 影响：
  - 可读性下降，尤其是小字号标签、辅助按钮、空状态文案。
  - 活跃态与普通态差距不足，用户不容易快速捕捉当前焦点。
  - 页面看起来精致，但可信度和操作把握感不足。
- 建议：
  - 重新定义一组更有梯度的中性色，至少把 `border`、`muted-foreground`、`secondary` 的层级拉开。
  - 减少在文字和图标上继续叠加 `/30`、`/40` 透明度的做法。
  - 为选中态、可点击态、空态分别建立明确 token，而不是都落到一组浅灰里。
- 优先级：高

## 问题 4：不同模块的排版语言差异过大，整体风格有“拼接感”
- 标题：同一产品里混用了至少三套视觉语法
- 证据（组件/页面/样式）：
  - `src/views/day/components/Sidebar.vue:25, 37` 使用倾斜、全大写、强 tracking 的海报风日期标题
  - `src/components/Navbar.vue:69, 109, 127` 导航项也采用 uppercase + 宽字距 + 胶囊悬浮风格
  - `src/views/habits/components/HabitSidebar.vue:13-17` 与 `src/views/direction/components/DirectionSidebar.vue:117-127` 则是常规 SaaS 样式
  - `src/views/summary/index.vue:49-79` 是标准表单卡片中台风
  - `src/views/login/index.vue:19-67` 又是最基础的朴素登录表单
- 问题：
  - 时序、习惯、所向、总结、登录五个模块的气质并不统一，像是分别来自不同阶段或不同模板。用户跨页面时会感觉“换了一个产品”。
- 原因：
  - 虽然项目有统一 theme token，但没有统一的版式规范和组件语言规范；各模块各自定义标题、表单、导航和卡片风格。
- 影响：
  - 品牌识别弱，产品成熟度感被削弱。
  - 用户需要在不同页面重新学习“这里什么是主要按钮、什么是标题、什么是辅助信息”。
  - 后续继续加页面时，风格漂移会越来越严重。
- 建议：
  - 先确定一条主视觉方向：如果保留“时序”的编辑杂志感，就要把 summary/login/habits 也统一到同一套标题、按钮、容器语法；如果偏效率工具，就要收敛 day/month 的海报化表达。
  - 建立最小设计规范：页面标题、分组标题、主按钮、次按钮、输入框、空状态、侧栏 item 的统一样式。
  - 登录页优先切换到项目通用 `Input` / `Button` / `Card` 体系，避免成为风格断点。
- 优先级：中

## 问题 5：方向页三栏布局节奏失衡，中间空白过大、右侧归档感知过弱
- 标题：方向页空间分配失衡，视觉重心漂浮
- 证据（组件/页面/样式）：
  - `src/views/direction/index.vue:39-49, 86-95` 固定左栏 + 中栏 `max-w-[600px]` + 右栏自适应
  - `src/views/direction/components/MissionBoard.vue:14-20` 主面板只有 `pb-32 px-6`
  - 截图 `screenshots/03-direction.png` 中，中间区域只有一张月份卡片，下面留下大面积空白；右侧“任务归档”虽占一整列，但内容层级很弱
- 问题：
  - 页面被强制切成三列，但当前信息量并没有撑起三列结构。中间规划区显得很空，右侧归档区又不够强，导致整页视觉中心漂浮在上半部分一小块卡片上。
- 原因：
  - 布局先决定了三栏，再往里填内容；而内容卡片的视觉重量不足，无法支撑这么大的版面。
- 影响：
  - 用户需要移动更长的视线距离来理解“左侧选目标 - 中间排月份 - 右侧看归档”的关系。
  - 空白不是在服务聚焦，而是在放大内容稀薄感。
- 建议：
  - 在信息量较低时改为 2 栏或 2.5 栏布局：归档区可折叠为抽屉或 tab。
  - 中栏的月份规划卡需要更强的标题区、说明区和操作区，避免像悬在空白中的孤立卡片。
  - 给右侧归档区更明确的容器头部、分组边界和状态色，让它在视觉上承担“结果区”的角色。
- 优先级：中

## 问题 6：表单控件和按钮在不同页面的尺寸、圆角和语义不统一
- 标题：输入与提交控件缺少统一的组件节奏
- 证据（组件/页面/样式）：
  - `src/components/ui/input/Input.vue` 全局输入框是 `h-9 rounded-md border-input`
  - `src/views/habits/components/HabitTodayCard.vue:37-56` 使用圆角卡片 + 圆形发送按钮 + 紧凑输入条
  - `src/views/login/index.vue:21-58` 直接手写一套输入框和按钮，圆角、边框、聚焦态与全站不同
  - `src/views/direction/components/MissionBoardMonthHeader.vue:7-39, 105-123` 月度规划输入区又是另一套尺寸与节奏
- 问题：
  - 同样是“输入并提交”，不同页面的控件尺寸、圆角、阴影、按钮形态差别很大。用户无法形成稳定的操作预期。
- 原因：
  - 项目虽然有基础 UI 组件，但页面仍在大量局部覆写或直接绕过组件体系。
- 影响：
  - 界面显得不够成体系，尤其登录页和业务页像来自两个系统。
  - 设计维护成本增加，后续要做统一改版时会牵扯大量页面特化样式。
- 建议：
  - 把登录页迁移到统一 UI 组件体系。
  - 为“主提交按钮”“图标按钮”“内联输入”“设置型输入”定义标准尺寸和圆角级别。
  - 页面允许做视觉强调，但基础控件的骨架不应每页重写。
- 优先级：中

## 总结
- 当前界面方向不是没有特色，问题主要在执行层，而不是方向层。项目已经有较明确的极简黑白基调和部分杂志化表达，但没有把这套语言稳定地落到所有页面。
- 最影响 UI 质量的不是“是否更好看”，而是三件事：
  - 信息层级把装饰性数字放得太大，业务信息反而变弱。
  - 交互入口隐藏过深，控件可识别性不足。
  - 各模块各自成风格，缺少统一的页面和控件语法。
- 如果只先改 3 件事，建议顺序是：
  - 先统一导航、列表编辑入口、按钮可见性，解决“看不出哪里能操作”。
  - 再收敛标题和数字层级，解决“视觉重点不等于任务重点”。
  - 最后统一登录页、总结页、习惯页和方向页的输入/按钮/卡片规范，消除拼接感。
