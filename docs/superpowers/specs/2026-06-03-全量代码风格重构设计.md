# 全量代码风格重构设计

## 背景

项目代码存在三类问题：注释过度冗余、移动端代码残留、组件/Composable 架构不合理。本设计按依赖链顺序分 4 个阶段一次性重构全部模块。

## 重构策略

**依赖链优先**：先清理噪音（注释 + 移动端），减少后续重构的代码量；再修架构（透传层 + 状态管理），由底向上。

**目标代码风格**：
- 注释只保留"为什么"和"不显然的事"，不解释代码本身已经表达的含义
- 子组件自己管理自己的 hooks，父组件只负责布局编排和页面级生命周期
- Pinia store 是状态的唯一真相源，不允许 composable 导出模块级 refs 做共享状态

## 阶段 1：注释精简

### 原则

| 保留 | 删除 |
|------|------|
| 解释非显而易见约束的行内注释 | 变量名已经说明含义的 JSDoc |
| 关键函数的 `@param` / `@returns`（参数含义不显然时） | 文件头的 `【模块职责】` 大段 HTML 注释 |
| 分组分隔线（`// ==== 状态 ====`） | 模板中重复模板结构的注释（`<!-- 搜索区开始 -->`） |
| `@example`（仅调用方式不显然时） | `<script>` 前的大段 JSDoc 描述块 |

### 注释标准示例

**好的注释：**
```js
// JavaScript 的 month 是 0-indexed，传入值需兼容
// 先移除旧监听器，防止快速 mousedown 叠加
```

**删除的注释：**
```js
/** 当前pending中的请求数量 */
const pendingCount = ref(0)

/**
 * ============================================
 * Day 视图 - 日时间轴主页面 (views/day/index.vue)
 * ============================================
 * 【模块职责】
 * - 日视图主页面容器
 * ...
 */
```

### 覆盖范围

全部 97 个 `.vue` 文件和 90 个 `.js` 文件，逐个清理。

## 阶段 3：删除 useDayData 透传层 + 子组件自管理 hooks

### 3a. 删除 `useDayData`

`useDayData.js` 只是 `dayStore` 的 bind 透传，直接删掉。所有引用处改为直接用 `dayStore`。

### 3b. 子组件自管理 hooks

| 组件 | 当前来源 | 重构后 |
|------|----------|--------|
| `Sidebar` | 父组件传 `isLoading`、`show` | 自己从 `dayStore` 取 `isLoading`，自己调 `useResizable` |
| `Timeline` | 父组件传 `currentHour`、`isLoading` | 自己从 `useDayNavigation` 取 |
| `AddEventModal` | 父组件传 `initialData` | 保留 prop（业务数据，非 hook 透传） |

**父组件 `day/index.vue` 瘦身后只负责：**
- 布局编排（Sidebar + Timeline + Modal 的位置关系）
- 页面级生命周期（通知权限、日报检查）
- Modal 显隐控制（`useDayModal` 留在父组件，多个 Modal 共享编辑状态）

## 阶段 4：修复 Direction 模块 Pinia 架构

### 问题

`useDirectionState.js` 导出模块级 refs，`directionStore.js` 只是 re-export facade。Pinia 不是状态真相源。

### 修复

1. 把 `useDirectionState.js` 中的所有状态定义搬入 `directionStore.js`
2. 删除 `useDirectionState.js`
3. 所有引用 `useDirectionState` 的 composable 改为从 `directionStore` 取
4. `months` 常量保留在 `directionStore.js` 中

### 影响文件

- `src/views/direction/composables/useDirectionState.js` — 删除
- `src/stores/directionStore.js` — 重写，吸收全部状态
- `src/views/direction/composables/useDirectionFetch.js` — 改 import 来源
- 其他引用 `useDirectionState` 的文件 — 改 import 来源
