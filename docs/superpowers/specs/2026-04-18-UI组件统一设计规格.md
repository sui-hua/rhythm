# UI 组件统一设计规格

> **目标**：全面替换硬编码颜色为 Semantic Tokens，移除所有 `dark:` 变体，统一登录页使用项目组件

## 1. 问题现状

### 1.1 硬编码颜色问题

项目大量使用 Tailwind 硬编码颜色（如 `bg-zinc-50`、`text-zinc-500`、`border-zinc-200`），而非项目的设计 token（semantic tokens）。这导致：
- dark mode 支持不完整，需要手动添加 `dark:` 变体
- 主题切换能力受限
- 代码可读性差

### 1.2 登录页组件问题

`src/views/login/index.vue` 使用原生 HTML `<input>` 和 `<button>`，未使用项目的 `Input.vue` 和 `Button.vue` 组件。

### 1.3 Badge outline 问题

`src/components/ui/badge/Badge.vue` 的 `outline` variant 缺少 dark mode 样式。

## 2. 设计 Token 映射规则

### 2.1 颜色替换对照表

| 当前硬编码 | 替换为 | 说明 |
|-----------|--------|------|
| `bg-zinc-50` | `bg-muted` | 静音背景 |
| `bg-zinc-100` | `bg-secondary` | 次要背景 |
| `bg-zinc-50/50` | `bg-muted/50` | 带透明度的静音背景 |
| `bg-zinc-50/80` | `bg-muted/80` | 带透明度的静音背景 |
| `bg-white` | `bg-background` 或 `bg-card` | 白色背景 |
| `bg-white/80` | `bg-background/80` 或 `bg-card/80` | 带透明度白色 |
| `border-zinc-100` | `border-border` | 边框 |
| `border-zinc-200` | `border-border` | 边框 |
| `border-zinc-300` | `border-border` | 边框（较深） |
| `text-zinc-500` | `text-muted-foreground` | 次要文本 |
| `text-zinc-600` | `text-foreground` | 主要文本 |
| `text-zinc-400` | `text-muted-foreground` | 静音文本 |
| `hover:bg-zinc-50` | `hover:bg-accent` | 悬停状态 |
| `hover:bg-zinc-100` | `hover:bg-accent` | 悬停状态 |
| `hover:border-zinc-300` | `hover:border-border` | 悬停边框 |
| `dark:bg-zinc-800` | **移除**，由 semantic token 自动响应 | — |
| `dark:bg-zinc-900` | **移除**，由 semantic token 自动响应 | — |
| `dark:border-white/10` | **移除**，由 semantic token 自动响应 | — |
| `dark:` 前缀 | **全部移除** | 依赖 semantic tokens |

### 2.2 Semantic Tokens 定义

项目在 `src/assets/tw-theme.css` 中定义：

```css
/* Light mode */
--background: oklch(1 0 0);
--foreground: oklch(0.141 0.005 285.823);
--muted: oklch(0.967 0.001 286.375);
--muted-foreground: oklch(0.552 0.016 285.938);
--accent: oklch(0.967 0.001 286.375);
--border: oklch(0.92 0.004 286.32);
--card: oklch(1 0 0);

/* Dark mode */
.dark {
  --background: oklch(0.141 0.005 285.823);
  --foreground: oklch(0.985 0 0);
  --muted: oklch(0.274 0.006 286.033);
  --muted-foreground: oklch(0.705 0.015 286.067);
  --accent: oklch(0.274 0.006 286.033);
  --border: oklch(1 0 0 / 10%);
  --card: oklch(0.21 0.006 285.885);
}
```

对应的 Tailwind 类：`bg-background`、`bg-muted`、`bg-accent`、`bg-card`、`text-foreground`、`text-muted-foreground`、`border-border`

## 4. 具体修改指南

### 4.1 移除 dark: 变体的判断原则

如果某元素使用了 semantic token（如 `bg-muted`），则：
- **移除** 所有 `dark:` 变体
- 理由：semantic token 在 `.dark` 模式下已自动切换

如果某元素需要**同时支持 light 和 dark**，且当前只有一种实现：
- 改用 semantic token（如 `bg-muted` 替代 `bg-zinc-50`）
- 移除对应的 `dark:` 变体

### 4.2 glass-card 特殊处理

```css
/* 当前 */
.glass-card {
  @apply bg-white/80 backdrop-blur-xl border border-white/20 shadow-[var(--shadow-whisper)];
}

.dark .glass-card {
  @apply bg-black/40 backdrop-blur-xl border-white/10;
}

/* 修改为 */
.glass-card {
  @apply bg-background/80 backdrop-blur-xl border border-border shadow-[var(--shadow-whisper)];
}
```

### 4.3 登录页改造

```html
<!-- 改造前 -->
<input class="w-full px-3 py-2 border border-zinc-200 rounded-md...">
<button class="w-full h-10 rounded-md bg-foreground text-background...">

<!-- 改造后 -->
<Input class="..." />
<Button class="...">
```

## 5. 实施注意事项

### 5.1 颜色替换优先级

1. **高优先级**：`bg-white`、`bg-zinc-50`、`border-zinc-*`、`text-zinc-500`
2. **中优先级**：`hover:bg-zinc-*`、带透明度的变体
3. **低优先级**：边界情况（如 `border-white/20`）

### 5.2 验证方法

修改完成后，通过以下方式验证：
1. 切换 dark mode，确认所有颜色正确切换
2. 检查是否有遗漏的硬编码颜色

### 5.3 不需要修改的情况

- 纯装饰性的 `text-zinc-300` 作为占位符（如果无语义含义）
- 图表相关的颜色（`chart-*`）
- 阴影相关的颜色（`shadow-*`）

## 6. Git 提交规则

根据项目规则：
- **Git 提交时机**：完成整个 UI 组件统一功能后再提交，不在功能进行中提交
- **Git 提交权限**：实现计划执行完成后，不执行 `git commit`，由用户自己提交

---

## 7. 预期收益

- **dark mode 完全由 semantic tokens 控制**，无需手动维护
- **主题切换能力**：未来换主题只需改 `tw-theme.css`
- **代码一致性**：所有组件使用统一的语义化颜色
- **登录页风格统一**：与项目其他页面保持一致

---

## 8. 风险评估

- **风险低**：纯替换，不改变布局和功能
- **建议**：分批次提交，便于回滚
