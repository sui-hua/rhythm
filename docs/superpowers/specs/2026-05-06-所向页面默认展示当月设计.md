# 所向页面默认展示当月功能设计

## 变更概述

**需求：** 所向页面（Direction）中间列表默认展示第一个月，改为默认展示当前月；当前月不在该目标的月度计划范围内时，继续展示排序后的第一个月。

**文件：** `src/views/direction/composables/useDirectionFetch.js`

## 现状逻辑

进入所向页面时，`fetchData()` 根据以下流程设置 `selectedMonth`：

1. 加载 `plans`（顶层目标）
2. 默认选中第一个目标
3. 加载该目标的 `monthlyPlans`
4. 按月份排序后选中第一个月
5. 加载该月的 `dailyPlans`

切换左侧目标时，`selectedGoal` 的 watch 使用同样的“排序后选第一个月”逻辑。

```javascript
// 当前代码（src/views/direction/composables/useDirectionFetch.js）
if (monthlyPlans.value.length > 0) {
  const sorted = [...monthlyPlans.value].sort((a, b) => {
    const mA = getDateOnlyMonth(a.month) || 0
    const mB = getDateOnlyMonth(b.month) || 0
    return mA - mB
  })
  const firstMp = sorted[0]
  if (firstMp) {
    const firstMonth = getDateOnlyMonth(firstMp.month)
    if (firstMonth) {
      selectedMonth.value = firstMonth
      await loadDailyPlans(firstMp.id, { force: true })
    }
  }
}
```

## 变更逻辑

新增本文件内部 helper，统一初始加载和目标切换时的默认月份选择：

1. 接收当前目标的 `monthlyPlans`
2. 过滤掉无法解析月份的记录
3. 按 `getDateOnlyMonth(mp.month)` 从小到大排序
4. 使用 `new Date().getMonth() + 1` 获取当前月
5. 当前月存在于列表中时，返回当前月对应的 `monthlyPlan`
6. 当前月不存在时，返回排序后的第一个 `monthlyPlan`
7. 空列表时返回 `null`

## 影响范围

- 影响 `src/views/direction/composables/useDirectionFetch.js` 的 `fetchData()` 初始加载逻辑。
- 影响 `src/views/direction/composables/useDirectionFetch.js` 的 `selectedGoal` watch 切换目标逻辑。
- 不新增数据库字段，不修改 Supabase 表结构，不改变写入流程。
- UI 行为变化限定在 Direction 页面：默认展开月份从“第一个月”变为“当前月优先，第一个月兜底”。

## 验证范围

- `src/views/direction/composables/__tests__/useDirectionFetch.spec.js` 增加当月命中和当月缺失的单元测试。
- 人工验证 Direction 页面初始进入和切换左侧目标后的默认展开月份。

## 回滚方案

删除新增 helper，恢复 `fetchData()` 与 `selectedGoal` watch 中原有“排序后选第一个月并加载 dailyPlans”的代码块。
