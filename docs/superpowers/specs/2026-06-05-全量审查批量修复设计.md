# 全量审查批量修复设计

基于 10 个子代理并行审查（产品/UX/UI/性能/前端架构/后端架构/数据架构/系统架构/全栈/辣评），共发现 70+ 个问题。本设计按业务模块组织，分 8 个阶段执行。

---

## 阶段划分与依赖关系

```
Phase 0: 数据库层修复（前置基础）
    ↓
Phase 1: 全局前端基础（设计令牌、Navbar、类型系统）
    ↓
Phase 2~7: 各业务模块（Day → Direction → Habits → Summary → Month/Year → Auth/Login）
```

- Phase 0 是所有后续阶段的前置（DB schema 必须先正确）
- Phase 1 中的设计令牌替换影响 Phase 2~7 的 UI 修复
- Phase 2~7 之间基本独立
- 每个阶段完成后同步更新 AGENTS.md
- 最后执行完整测试验证（`pnpm vue-tsc` + `pnpm build` + `pnpm test`）

## Phase 0: 数据库层修复

### 0.1 FK 约束补充

以下 5 张表的 `user_id` 列缺少外键约束到 `auth.users`：

```sql
ALTER TABLE task ADD CONSTRAINT task_user_id_fkey
  FOREIGN KEY (user_id) REFERENCES auth.users(id);
ALTER TABLE goal ADD CONSTRAINT goal_user_id_fkey
  FOREIGN KEY (user_id) REFERENCES auth.users(id);
ALTER TABLE goal_months ADD CONSTRAINT goal_months_user_id_fkey
  FOREIGN KEY (user_id) REFERENCES auth.users(id);
ALTER TABLE goal_days ADD CONSTRAINT goal_days_user_id_fkey
  FOREIGN KEY (user_id) REFERENCES auth.users(id);
ALTER TABLE habit ADD CONSTRAINT habit_user_id_fkey
  FOREIGN KEY (user_id) REFERENCES auth.users(id);
```

### 0.2 级联删除策略修复

```sql
-- goal_days 对 goal_months: NO ACTION → CASCADE
ALTER TABLE goal_days DROP CONSTRAINT goal_days_goal_month_id_fkey;
ALTER TABLE goal_days ADD CONSTRAINT goal_days_goal_month_id_fkey
  FOREIGN KEY (goal_month_id) REFERENCES goal_months(id) ON DELETE CASCADE;

-- goal 对 goal_categories: NO ACTION → SET NULL
ALTER TABLE goal DROP CONSTRAINT goal_category_id_fkey;
ALTER TABLE goal ADD CONSTRAINT goal_category_id_fkey
  FOREIGN KEY (category_id) REFERENCES goal_categories(id) ON DELETE SET NULL;
```

### 0.3 status 类型统一

`goal_days.status` 从 `smallint` 改为 `varchar`，与 `goal`/`goal_months` 一致：

```sql
-- 先添加新列
ALTER TABLE goal_days ADD COLUMN status_new varchar DEFAULT 'active';
-- 数据迁移
UPDATE goal_days SET status_new = CASE WHEN status = 0 THEN 'active' WHEN status = 1 THEN 'completed' ELSE 'active' END;
-- 删除旧列，重命名新列
ALTER TABLE goal_days DROP COLUMN status;
ALTER TABLE goal_days RENAME COLUMN status_new TO status;
-- 添加 CHECK 约束
ALTER TABLE goal_days ADD CONSTRAINT goal_days_status_check
  CHECK (status IN ('active', 'completed', 'archived'));
```

**前端同步**：删除 `toGoalDayStatus` 转换函数，直接使用字符串值。

### 0.4 updated_at 触发器

```sql
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- 对所有含 updated_at 的表创建触发器
CREATE TRIGGER set_updated_at BEFORE UPDATE ON task
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_updated_at BEFORE UPDATE ON goal
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_updated_at BEFORE UPDATE ON goal_months
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_updated_at BEFORE UPDATE ON goal_days
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_updated_at BEFORE UPDATE ON habit
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_updated_at BEFORE UPDATE ON habit_logs
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
CREATE TRIGGER set_updated_at BEFORE UPDATE ON summary
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### 0.5 索引优化

```sql
-- 删除冗余索引（task.start_time 已被复合索引前缀覆盖）
DROP INDEX IF EXISTS idx_tasks_start_time;

-- 新增复合索引
CREATE INDEX idx_task_user_start ON task(user_id, start_time);
CREATE INDEX idx_habit_logs_completed ON habit_logs(habit_id, completed_at);
CREATE INDEX idx_goal_days_user_day ON goal_days(user_id, day);
```

### 0.6 habit.log() user_id 自动填充

```sql
-- 设置 habit_logs.user_id 默认值为当前认证用户
ALTER TABLE habit_logs ALTER COLUMN user_id SET DEFAULT auth.uid();
```

### 0.7 summary.kind CHECK 约束

```sql
ALTER TABLE summary
  ADD CONSTRAINT summary_kind_check
  CHECK (kind IN ('daily', 'weekly', 'monthly', 'yearly'));
```

### 验证

- 执行 `get_advisors(type: 'security')` 和 `get_advisors(type: 'performance')` 检查无新增警告
- `list_tables` 确认所有约束和索引生效

```typescript
export async function safeAction<T>(
  action: () => Promise<T>,
  errorMessage: string
): Promise<T | null> {
  try {
    return await action()
  } catch (e) {
    console.error(errorMessage, e)
    // 触发 toast 通知
    return null
  }
}
```
