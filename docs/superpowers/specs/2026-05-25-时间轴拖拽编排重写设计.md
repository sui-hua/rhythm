# 时间轴拖拽编排重写设计

For agentic workers: 必须严格按照本设计执行，不得擅自扩展到触屏、Habit、DailyPlan 或跨天拖拽。

## Goal

重写 Day 页面时间轴中的任务拖拽编排功能，目标明确收口为桌面端鼠标交互：

- 仅支持 `task.type === 'task'` 且未完成任务
- 支持纵向拖拽移动开始时间
- 支持底部缩放调整结束时间
- 拖拽过程中卡片实时跟手
- 时间吸附粒度固定为 5 分钟
- 仅在 `mouseup` 时写库一次
- 写库失败时卡片回弹到原始位置

## Architecture

### 核心拆分

| 文件 | 职责 |
|------|------|
| `src/views/day/components/TaskItem.vue` | 只负责卡片内容展示，不承担拖拽定位和会话状态 |
| `src/views/day/components/TaskItemWrapper.vue` | 负责事件接线、wrapper 几何样式、tooltip 渲染、提交触发 |
| `src/views/day/composables/useTimelineDragSession.js` | 新增，负责 drag / resize 会话状态、draft 几何值、吸附、边界裁剪、回弹 |
| `src/views/day/composables/useDragSnap.js` | 保持纯计算层，负责像素与时间转换、吸附和边界约束 |
| `src/views/day/utils/taskLayoutStyle.js` | 只负责横向布局规则：单列铺满、多列分栏、carry-over 堆叠 |

### 组件层级

```text
Timeline.vue
  └─ TaskItemWrapper.vue
       ├─ useTimelineDragSession()
       ├─ TaskItem.vue
       └─ DragTimeTooltip.vue
```

### 数据流

```text
mousedown
  → useTimelineDragSession.startDrag / startResize
  → document mousemove
  → 计算 draftStartHour / draftDurationHours
  → TaskItemWrapper 使用 draft 几何值实时渲染
  → document mouseup
  → useTimelineDragSession.finish()
  → useDayData.updateTaskTime()
  → store.fetchTasks() 刷新真实数据

若 finish() 失败
  → 丢弃 draft 状态
  → wrapper 回到 props 真值
```

## Tech Stack

- Vue 3 Composition API + `<script setup>`
- 现有 `useDayData` / `dayStore` 数据链路
- 原生 `mousedown` / `mousemove` / `mouseup`
- 现有 Vitest 单测体系

## 设计细节

### 1. 交互边界

本次重写只覆盖桌面端鼠标场景：

- 不接入 Pointer Events
- 不兼容移动端触摸拖拽
- 不支持 Habit、Goal Day 的拖拽和缩放
- 不支持水平拖拽
- 不支持跨天编排
- 不支持拖拽中持续写库

### 2. `useTimelineDragSession` 的状态模型

新增 composable 统一管理一次拖拽或缩放会话，内部至少包含以下状态：

| 状态 | 含义 |
|------|------|
| `mode` | `idle` / `drag` / `resize` |
| `startClientY` | 本次按下时的鼠标 Y 坐标 |
| `baseStartHour` | 本次会话冻结的原始开始时间 |
| `baseDurationHours` | 本次会话冻结的原始时长 |
| `draftStartHour` | 实时预览中的开始时间 |
| `draftDurationHours` | 实时预览中的时长 |
| `isSaving` | `mouseup` 后是否正在提交 |
| `lastError` | 最近一次保存失败信息，仅供调试和日志 |

设计约束：

- `base*` 只在 `mousedown` 时写入一次
- `draft*` 只在会话进行中更新
- 组件渲染优先使用 `draft*`，没有会话时回退到 `task.startHour / task.durationHours`
- `mouseup` 后只触发一次提交
- 提交失败必须清空 `draft*`，让 UI 自动回到 props 真值

### 3. `useTimelineDragSession` 的公开接口

建议公开以下接口，保持组件层最小化：

```js
{
  isActive,
  isDragging,
  isResizing,
  isSaving,
  draftStartHour,
  draftDurationHours,
  tooltipHour,
  tooltipDuration,
  startDrag(mouseEvent),
  startResize(mouseEvent),
  updateFromMouse(mouseEvent),
  finish(),
  cancel()
}
```

语义要求：

- `startDrag`：初始化 drag 会话
- `startResize`：初始化 resize 会话
- `updateFromMouse`：由全局 `mousemove` 驱动，统一更新 draft 值
- `finish`：由全局 `mouseup` 调用，输出最终 snapped 结果并提交
- `cancel`：组件卸载或异常退出时清理监听和 draft

### 4. draft 几何渲染规则

`TaskItemWrapper` 的纵向样式改为由“当前显示值”统一推导：

- `displayStartHour = draftStartHour ?? task.startHour`
- `displayDurationHours = draftDurationHours ?? task.durationHours`
- `top = hourToPx(displayStartHour)`
- `height = hourToPx(displayDurationHours)`

这样拖拽中的卡片位置与 tooltip 共享同一套状态，不再出现“tooltip 更新但卡片不动”的分叉。

横向样式继续来自 `taskLayoutStyle.js`：

- 单列任务铺满
- 多列并发任务按 `_col / _numCols` 分栏
- 只有 `_stackSize > 1` 的 carry-over 才走堆叠样式

### 5. 吸附与边界统一规则

`useDragSnap.js` 保持纯函数，并补齐给 `useTimelineDragSession` 复用的入口。

统一规则：

- 吸附粒度固定为 5 分钟
- 拖拽时保持时长不变，只改开始时间
- 缩放时保持开始时间不变，只改结束时间
- 开始时间下界为 `0`
- 结束时间上界为 `24`
- 最小时长为 `5 / 60`

需要避免两套计算并存：

- 拖拽预览使用一套纯函数
- `mouseup` 提交使用同一套纯函数

禁止继续保留“tooltip 用一套计算、持久化用另一套计算”的实现。

### 6. 提交流程

`mouseup` 时的处理顺序固定为：

1. 移除全局事件监听
2. 判断本次 draft 是否与 base 不同
3. 如果没有变化，直接清理会话并退出
4. 如果有变化，调用 `updateTaskTime(task, newStartHour, newEndHour)`
5. 成功后清理会话，等待 store 刷新 props
6. 失败后清理 draft，卡片回到原位置

设计决定：

- 不在本地做“提交成功后强行写死位置”的额外同步层
- 以 store 刷新后的 props 为最终真值
- 提交失败只回弹，不做局部脏状态保留

### 7. `TaskItemWrapper.vue` 的职责边界

重写后 `TaskItemWrapper.vue` 只保留以下职责：

- 判断 `canDrag`
- 调用 `useTimelineDragSession`
- 绑定卡片主体的 `mousedown` 和底部手柄的 `mousedown`
- 绑定 document 级 `mousemove / mouseup`
- 根据 draft 状态生成 wrapper 的 `top / height`
- 渲染 `DragTimeTooltip`
- 在 `finish()` 阶段调用 `updateTaskTime`

以下职责必须从组件中移出：

- 拖拽公式本体
- 吸附和边界裁剪公式
- “当前是否显示 draft” 的分叉判断散落在多个地方

## 不做的事

- 不恢复 `vue3-draggable-resizable`
- 不支持触屏和 Pointer Events
- 不支持 Habit / Goal Day 拖拽
- 不支持跨天或横向拖拽
- 不支持拖拽时自动滚动
- 不引入新的全局状态管理层

## 风险与对应策略

| 风险 | 策略 |
|------|------|
| draft 与 props 混用导致位置跳动 | 所有纵向渲染统一读 `displayStartHour/displayDurationHours` |
| tooltip 与卡片位置不一致 | tooltip 与 wrapper 共用同一套 draft 值 |
| `mouseup` 后重复提交 | `finish()` 内部只允许一次有效提交 |
| 卸载后遗留全局监听 | `cancel()` 负责统一清理 document 监听 |
| 单个 carry-over 再次变窄 | `taskLayoutStyle.spec.js` 固化该回归用例 |
