# 全量代码质量修复设计

## 背景

对整个代码库进行质量审查，发现 20 个问题（8 HIGH / 9 MEDIUM / 3 LOW），涉及安全漏洞、功能 bug、响应性丢失、资源泄漏等。本设计按依赖链优先策略，分 6 个阶段一次性修复全部问题。

## 修复策略

**依赖链优先**：先修底层基础（supabase 配置、stores 响应性），再修上层消费方，避免因基础问题导致返工。

**验证方式**：修复完成后运行 `pnpm test:unit` + `pnpm build` 确认不破坏现有功能。

---

## 阶段 1：底层基础

### 1.1 `src/services/supabase.js`

**问题 A**：环境变量缺失时仅 console.warn，Supabase client 仍用 undefined 创建，运行时报 cryptic 错误。
**修复**：warn 后 throw Error，阻止应用启动。

**问题 B**：anon key 被完整打印到控制台。
**修复**：日志中遮蔽 key，仅显示前 4 位 + `***`。

### 1.2 `src/stores/pageStateStore.js`

**问题**：`...state` 展开 reactive 对象，day/summary/direction 变成普通对象，修改不触发更新。
**修复**：移除展开，直接返回 `state.day`、`state.summary`、`state.direction` 保持响应性。

### 1.3 `src/stores/pomodoroStore.js`

**问题**：setInterval 从未清理，软导航后泄漏。
**修复**：暴露 `stop()` 方法清理定时器；在 store 初始化时注册 `$dispose` 回调。

### 1.4 `src/stores/dayStore.js`

**问题**：`useRoute()` 在 store 定义顶层调用，组件外访问会抛异常。
**修复**：将 `useRoute()` 调用移入需要它的 action 内部。

## 阶段 2：Composables 层

### 2.1 `src/composables/useNotifications.js`

**问题 A**：`notifiedTaskIds` 是模块级单例，`clearNotifiedHistory()` 会影响所有组件。
**修复**：保持模块级（通知本身就是全局单例），但在 `clearNotifiedHistory` 调用处添加注释说明其全局影响，限制调用场景。

**问题 B**：`startListening` 无重入保护，多次调用会叠加轮询。
**修复**：添加 `isListening` 标志，重复调用时跳过。

### 2.2 `src/composables/useResizable.js`

**问题**：快速 mousedown 叠加注册 mousemove/mouseup 监听器。
**修复**：`startResize` 开头先 `removeEventListener` 移除旧监听器，再添加新的。

### 2.3 `src/composables/useGlobalLoading.js`

**问题**：showTimer/hideTimer 未在 app teardown 时清理。
**修复**：导出 `cleanup()` 函数，清除两个定时器并重置状态。

### 2.4 `src/stores/onboardingStore.js`

**问题**：`activeStep` 硬编码为第一步，刷新后 completedSteps 恢复但 activeStep 重置。
**修复**：在 store 初始化时，遍历 `defaultSteps` 找到第一个不在 `completedSteps` 中的步骤作为 `activeStep`；若全部完成则保持最后一步。

## 阶段 3：Day 视图

### 3.1 数组索引稳定性

**`useDayModal.js`**：`editingTask` 存储任务 id 而非数组下标，computed 通过 id 查找。
**`Sidebar.vue`**：滑动完成回调改为接收任务对象，不依赖数组位置。

### 3.2 `TaskItemWrapper.vue` 事件监听器去重

拖拽开始时检查是否已绑定（布尔标志），有则跳过；拖拽结束时统一移除并重置标志。

### 3.3 `Sidebar.vue` `_originalIndex`

在 `buildSidebarSections` 内统一计算 `_originalIndex`，不依赖两处 computed 的顺序一致性。

### 3.4 `PomodoroTimerModal.vue` 圆环

circumference 从硬编码 290 改为 `2 * Math.PI * 46`（≈289.03）。

### 3.5 `useAddEventForm.js` 时间校验

split `:` 后校验 hours ∈ [0,23]、minutes ∈ [0,59]，非法则拒绝提交并提示。

### 3.6 未使用 import 清理

- `TaskItem.vue`：删除 CheckCircle
- `PomodoroTimerModal.vue`：删除 Badge
- `MobileAddEventDrawer.vue`：删除 formatDuration

### 4.1 `HabitMobileListSheet.vue` isTodayCompleted()

**修复**：改为 `completedDays.includes(today.getDate())`。

### 4.2 频率表单逻辑去重

`AddHabitModal.vue` 和 `EditHabitModal.vue` 中 WEEKDAY_OPTIONS、MONTH_DAY_OPTIONS、canSubmitFrequency、toggleWeekday、toggleMonthDay、buildFrequencyPayload 完全重复。
**修复**：提取到 `src/views/habits/composables/useHabitFrequencyForm.js`，两处 import。

### 4.3 `DirectionSidebar.vue` 未使用变量

删除未读取的 `monthPlan` 及其 find 调用。

### 4.4 `MissionBoardMonthHeader.vue` 月份显示

10/11/12 月不补零，避免 `010月`。改为条件补零：`month < 10 ? '0' + month : month`。
