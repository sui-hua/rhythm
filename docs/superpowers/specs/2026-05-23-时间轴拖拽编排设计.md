# 时间轴拖拽编排设计

For agentic workers: 必须严格按照本设计执行，不得擅自修改设计内容。

## Goal

在日视图（Day View）时间轴上实现任务卡片的拖拽移动和底部缩放调整时长功能，使用户可以通过拖拽直观地重新安排任务时间，无需打开编辑弹窗。

## Architecture

### 新增文件

| 文件 | 职责 |
|------|------|
| `src/views/day/components/TaskItemWrapper.vue` | 包裹 VDR + DragTimeTooltip 的容器，处理拖拽/缩放事件 |
| `src/views/day/components/DragTimeTooltip.vue` | 拖拽时的时间提示浮层 |
| `src/views/day/composables/useDragSnap.js` | 像素↔时间转换 + snap 逻辑，纯函数 |

### 改动文件

| 文件 | 改动点 |
|------|--------|
| `src/views/day/components/Timeline.vue` | 将 `<TaskItem>` 替换为 `<TaskItemWrapper>` |
| `src/views/day/composables/useDayData.js` | 新增 `updateTaskTime(task, newStartHour, newEndHour)` 方法 |

### 组件层级

```
Timeline.vue
  └─ TaskItemWrapper.vue (新增)
       ├─ VueDraggableResizable  ← 拖拽/缩放引擎（vue3-draggable-resizable）
       │    └─ TaskItem.vue      ← 现有卡片，内部逻辑不变
       └─ DragTimeTooltip.vue    (新增) 拖拽时的时间提示
```

### 数据流

```
用户拖拽 → VDR 组件更新 x/y/w/h → onDragEnd / onResizeEnd
  → useDragSnap: 像素→时间（5分钟对齐）
    → useDayData.updateTaskTime(): 写入 DB
      → fetchTasks(): 刷新数据
```

## 设计细节

### 1. useDragSnap.js — 像素↔时间转换

纯函数 composable，提供以下能力：

```
常量：
  HOUR_HEIGHT = 180  (px，对应 CSS max(25vh, 180px) 的最小值)
  SNAP_MINUTES = 5
  SNAP_PX = 15       (5分钟 = 15px)

函数：
  pxToHour(px) → number     像素→小时，snap 到 5 分钟
  hourToPx(hour) → number   小时→像素
  calcDragResult(task, deltaY) → { newStart, newEnd }
    拖拽移动：保持 duration，计算新 startHour
    边界约束：0 ≤ startHour ≤ 24 - durationHours
  calcResizeResult(task, newHeight) → { newStart, newEnd }
    缩放：保持 startHour，计算新 duration
    边界约束：最小 5 分钟，最大 24 - startHour
```

### 2. TaskItemWrapper.vue — 拖拽容器

职责：
- 判断是否可拖拽：`canDrag = !task.completed && task.type === 'task'`
- 不可拖拽时直接渲染 `<TaskItem>`，不包裹 VDR
- 可拖拽时包裹 VDR，处理拖拽/缩放事件
- 管理拖拽状态（isDragging），驱动 Tooltip 显示

VDR 配置：
```
axis="y"              仅纵向拖拽
:w="-1"               满宽，不可水平拖
:snap="{ x: 0, y: SNAP_PX }"  5分钟吸附
:parent="true"        限制在父容器内
:draggable="canDrag"  不可拖拽时禁用
:resizable="canDrag"  不可缩放时禁用
:minHeight="SNAP_PX"  最小高度 = 5分钟
```

拖拽中的视觉反馈：
- 卡片加 `opacity-70 shadow-lg scale-[1.02]`，视觉上"浮起来"
- 通过 `@drag-start` / `@drag-end` 切换 class

### 3. DragTimeTooltip.vue — 时间提示浮层

- 位置：固定在卡片右侧，超出视口时切换到左侧
- 内容：`HH:mm - HH:mm` 格式，拖拽过程中实时更新
- 显示条件：`isDragging === true` 时显示，松手后 200ms 淡出
- 样式：半透明深色背景 + 白色文字 + 小箭头
- 使用 Teleport 渲染到 body，不受父组件 z-index 影响

### 4. useDayData.js — 新增 updateTaskTime

```js
const updateTaskTime = async (task, newStartHour, newEndHour) => {
  const { year, month, day } = routeDateContext.value
  const baseDate = new Date(year, month - 1, day)

  const startTime = new Date(baseDate)
  startTime.setHours(Math.floor(newStartHour), (newStartHour % 1) * 60)

  const endTime = new Date(baseDate)
  endTime.setHours(Math.floor(newEndHour), (newEndHour % 1) * 60)

  await db.tasks.update(task.id, {
    start_time: startTime.toISOString(),
    end_time: endTime.toISOString()
  })

  await fetchTasks({ showLoading: false })
}
```

## 约束与边界

| 场景 | 处理方式 |
|------|----------|
| 已完成任务 | `draggable=false` + `resizable=false`，无 cursor 变化 |
| Habit / DailyPlan | 不包裹 VDR，直接渲染 TaskItem |
| 拖拽超出当天边界 | startHour 夹紧到 `[0, 24 - duration]` |
| 缩放最小值 | 5 分钟（1 个 snap 单位） |
| 缩放最大值 | `24 - startHour` |
| DB 写入失败 | toast 提示，不回滚 UI，fetchTasks 刷新真实状态 |
| 滚动联动 | 拖拽到视口顶部/底部边缘时自动滚动 ScrollArea |

## 不做的事

- 不支持移动端触控拖拽（后续迭代）
- 不支持 Habit / DailyPlan 拖拽
- 不支持水平拖拽（跨天/跨列）
- 不做拖拽时的时间轴辅助线
