# Day 页面端到端测试 + 体验改进计划

## 目标

1. 使用 Playwright MCP 对 Day 页面进行全面端到端功能测试
2. 从用户角度深度分析体验改进点，输出优化建议

---

## Part 1: 端到端功能测试

### 测试范围

#### 1. 任务（Task）CRUD

| 测试项 | 操作 | 验证点 |
|--------|------|--------|
| 创建任务-正常 | 点击添加 → 填写表单 → 保存 | 任务出现在时间轴和侧边栏 |
| 创建任务-设置时长 | 创建时指定 2 小时时长 | 显示时长正确 |
| 编辑任务-标题 | 双击任务 → 修改标题 → 保存 | 标题更新 |
| 编辑任务-时间 | 双击任务 → 修改时间 → 保存 | 时间更新，无 Invalid time value |
| 编辑任务-时长 | 双击任务 → 修改时长 → 保存 | 时长更新 |
| 删除任务 | 双击任务 → 删除 | 任务消失 |
| 完成任务 | 点击任务左侧复选框 | 任务显示完成状态 |

#### 2. 日计划（Daily Plan）

| 测试项 | 操作 | 验证点 |
|--------|------|--------|
| 创建日计划 | 侧边栏添加 → 选择日计划类型 | 日计划创建成功 |
| 编辑日计划 | 双击日计划 → 修改 → 保存 | 日计划更新 |
| 完成日计划 | 点击复选框 | 状态变为已完成 |

#### 3. 习惯（Habits）

| 测试项 | 操作 | 验证点 |
|--------|------|--------|
| 查看今日习惯 | 导航到 day 页面 | 习惯列表显示 |
| 习惯打卡 | 点击习惯复选框 | 习惯标记为已完成 |
| 取消打卡 | 再次点击已打卡习惯 | 恢复未完成状态 |

#### 4. 边界情况验证

| 测试项 | 操作 | 验证点 |
|--------|------|--------|
| 空时间字段 | 编辑任务时清空时间 → 保存 | 正确处理，不报 Invalid time value |
| 无效时间格式 | 数据库时间解析 | 控制台无 Error |
| 时间解析验证 | 检查 useDayData 解析 start_time | 有效时间正确解析为 HH:mm |

### 测试环境

- **URL**: `http://localhost:5173`
- **账号**: `123456@163.com`
- **密码**: `123456`

### 测试方法

使用 Playwright MCP 工具：
- `browser_navigate` - 页面导航
- `browser_snapshot` - 获取页面结构
- `browser_click` - 点击操作
- `browser_fill_form` - 表单填写
- `browser_type` - 文本输入
- `browser_take_screenshot` - 截图留证
- `browser_console_messages` - 检查控制台错误

### 通过标准

- [ ] 页面正常加载，无控制台 Error
- [ ] 任务创建、编辑、删除正常
- [ ] 编辑任务时无 `Invalid time value` 错误
- [ ] 日计划创建和管理正常
- [ ] 习惯打卡功能正常
- [ ] 所有数据正确保存到数据库
