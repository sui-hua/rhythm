# 数据库清理设计方案

## 概要

对齐数据库中与实际代码不符的结构问题，清理废弃表、冗余列、错误约束和不一致的兼容层。

## 问题清单

| # | 问题 | 对象 | 说明 |
|---|------|------|------|
| 1 | `scope` 已废弃仍需强制写入 | `summaries` | kind 已完全替代 scope，但 scope 仍有 NOT NULL + check 约束 |
| 2 | `mood` 从未使用 | `summaries` | 3 条数据 mood 全 null，用户确认删除 |
| 3 | `daily_summaries` 废弃表 | `daily_summaries` | 整表 0 数据，代码已迁移至 summaries |
| 4 | `archived` / `is_archived` 重复 | `habits` | 两列语义重复，9 条数据值不一致，is_archived 是真实数据 |
| 5 | 日总结数据不规整 | `summaries` | 3 条日总结 scope=`week`，实为废弃兼容逻辑所致 |

## 排除范围

- `daily_plans.status` 类型不一致（smallint 0/1 vs varchar active/completed/archived）—— 不影响功能，本次不处理
- `tasks.start_time/end_time` 和 `actual_start_time/actual_end_time` 并存 —— 语义不同（计划 vs 实际执行），有意为之
- `batch_upsert_daily_plans` / `batch_delete_daily_plans` RPC —— 当前运行代码未调用，实际使用逐条 CRUD，本次不创建死函数

## 执行步骤

### Step A：`summaries` 表整理

**SQL 变更：**

```sql
-- 1. 删除 mood 列及 check 约束
ALTER TABLE summaries DROP COLUMN IF EXISTS mood;

-- 2. scope 列改为可空，移除 check 约束
ALTER TABLE summaries ALTER COLUMN scope DROP NOT NULL;
ALTER TABLE summaries DROP CONSTRAINT IF EXISTS summaries_scope_check;

-- 3. 修正已有数据的 scope
UPDATE summaries SET scope = NULL WHERE scope IS NOT NULL;
```

**代码清理：**

- `src/services/db/summaries.js`：删除 `bridgeLegacyScope()`、`normalizeScope()`、`kindToLegacyScope`、`allowedScopes` 整段逻辑（约 40 行）
- `src/views/summary/composables/useDailySummaryForm.js`：从 `formData` 和 `buildPayload` 中移除 mood
- `src/views/summary/components/DailySummaryForm.vue`：移除心情输入框
- `src/views/summary/composables/useGenericSummaryForm.js`：从 formData 和 `buildPayload` 中移除 mood
- `src/views/summary/components/GenericSummaryForm.vue`：移除心情输入框
- `src/views/summary/utils/summaryAdapters.js`：删除 `normalizeMood()`，`buildSummaryPayload` 移除 `normalizeMood(formData.mood)`
- 所有引用了 `mood` 的测试文件同步更新：
  - `src/views/summary/composables/__tests__/summaryAdapters.spec.js` — 删除 mood 相关测试
  - `src/services/db/__tests__/summaries.spec.js` — 删除 `scope: 'week'` 断言，改用新结构

### Step B：`habits` 表整理

**SQL 变更：**

```sql
ALTER TABLE habits DROP COLUMN IF EXISTS archived;
```

代码层无需变更，已全部使用 `is_archived`。

### Step C：废弃表

**SQL 变更：**

```sql
-- 删除废弃的 daily_summaries 表及关联 RLS 策略
DROP TABLE IF EXISTS daily_summaries CASCADE;
```

### Step D：代码清理

- `SummaryLaunchCard.vue`：删除未使用的组件文件
- `GenericSummaryForm.vue` `typeName` 计算属性中删除 `'day'` 死代码分支

## 测试要点

- 保存日总结后数据库 `summaries` 表中没有 mood 值
- 习惯列表/归档功能行为不变
- 已有总结列表正常加载渲染

## 回滚方案

所有 SQL 变更均为可逆：
- 删列：可重新添加
- 改约束：可重新加回
- 删表：从备份恢复
- 代码变更：git revert

建议执行前备份 `summaries` 和 `habits` 表数据。
