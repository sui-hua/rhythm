# 审查问题修复设计

基于项目全面审查，修复 20 个已确认问题，分 3 批执行。

---

## 批次 1：安全 + 架构基础

### 1.1 useDirectionFetch 模块级状态泄漏

**文件**: `src/views/direction/composables/useDirectionFetch.js`

**问题**: `isInitializing`、`hooksSetup`、`isPageLoading` 声明在函数外部，创建伪单例。`hooksSetup` 为 `true` 后后续调用跳过初始化，watch 无清理导致内存泄漏。

**修复**:
- 将 `isInitializing`、`hooksSetup`、`isPageLoading` 移入 `useDirectionFetch()` 函数体内
- 用 `onScopeDispose` 清理 watcher
- 移除 `hooksSetup` 标志位，每次调用都重新注册 watcher

### 1.2 .env 安全 + 访客凭据硬编码

**文件**: `.env`、`.gitignore`、`src/views/login/index.vue`

**修复**:
- 将 `.env` 加入 `.gitignore`
- 创建 `.env.example` 模板（只含变量名，不含值）
- 登录页访客账号区域改为"使用演示账号体验"按钮，点击自动填充并登录，不暴露密码明文

### 1.3 删除 withUserScope.js

**文件**: `src/services/db/withUserScope.js`

**修复**:
- 确认 Supabase RLS 策略已充分覆盖所有表
- 删除 `withUserScope.js`
- 删除 `database.js` 或 `safeDb.js` 中对它的引用（如有）

### 1.4 useDailyReport / useNotifications 模块级 ref

**文件**:
- `src/views/day/composables/useDailyReport.js`
- `src/composables/useNotifications.js`

**修复**:
- `useDailyReport.js`: 将 `reportVisible`、`reportStats`、`isLoading` 移入函数内部
- `useNotifications.js`: 将 `notifiedTaskIds` 移入函数内部；如需跨组件共享，改用 Pinia store

### 1.5 pomodoroStore setInterval 泄漏

**文件**: `src/stores/pomodoroStore.js`

**修复**:
- 在 `startTimer()` 中使用 `onScopeDispose` 注册清理回调
- 或将 timer 逻辑移到使用该功能的组件 composable 中，由组件生命周期管理

## 批次 2：性能优化

### 2.1 dayStore 全量请求拆分

**文件**: `src/stores/dayStore.js`

**问题**: `handleToggleComplete`、`handleStartTask`、`updateTaskTime` 都调用完整 `fetchTasks`（4 个并行请求）。

**修复**:
- 新增 `fetchTaskUpdate(taskId)` 方法，只拉取单条任务最新状态
- `handleToggleComplete` 改为乐观更新：先本地修改 `completed` 状态，调用 `fetchTaskUpdate` 同步，失败时回滚
- `handleStartTask` 和 `updateTaskTime` 同理

### 2.2 创建 habitStore + 习惯打卡乐观更新

**新文件**: `src/stores/habitStore.js`

**修改文件**:
- `src/views/habits/composables/useHabitData.js` → 改为从 habitStore 读取
- `src/views/habits/composables/useHabitLogs.js` → 打卡用 `buildPatchedHabit` 本地更新
- `src/views/day/composables/useDayExecutionItems.js` → 从 habitStore 读取
- `src/views/year/composables/useYearView.js` → 从 habitStore 读取

**habitStore 设计**:
```
state:
  allHabits: []          // 所有习惯列表
  selectedHabitId: null  // 当前选中习惯
  loading: false

actions:
  fetchHabits()          // 全量加载
  patchHabit(id, patch)  // 乐观更新单个习惯
  addHabit(habit)        // 新增
  updateHabit(id, data)  // 编辑
  archiveHabit(id)       // 归档
```

### 2.3 移除 radix-vue 重复依赖

**文件**: `package.json`、`src/components/ui/` 下各组件

**修复**:
- 检查 `src/components/ui/` 下各组件的 import 来源，确认使用的是 `reka-ui` 还是 `radix-vue`
- 移除未使用的那个依赖
- 通常应保留 `reka-ui`（下一代版本），移除 `radix-vue`

### 2.4 vueDevTools 条件加载

**文件**: `vite.config.js`

**修复**:
```js
plugins: [
  vue(),
  ...(process.env.NODE_ENV !== 'production' ? [vueDevTools()] : []),
  tailwindcss()
]
```

### 2.5 通知轮询频率优化

**文件**: `src/composables/useNotifications.js`

**修复**:
- `setInterval` 从 30000ms 改为 60000ms
- 增加 `lastCheckedMinute` 变量，避免同一分钟内重复检查
