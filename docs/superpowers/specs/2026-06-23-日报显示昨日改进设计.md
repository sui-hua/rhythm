# 日报显示昨日改进设计

## 背景

用户在 Summary 的日总结中会填写“改进之处”，这些内容目前只沉淀在总结记录里。进入第二天的 Day 页面时，每日日报弹窗已经承担“昨天复盘 · 今天计划”的入口，但只展示任务统计和顺延数量，没有把昨天总结中的改进事项带回到今天的执行场景。

本次设计目标是在今天进入日视图时弹出的每日日报弹窗中，展示昨天日总结里填写的“改进之处”，让昨天的反思变成今天可见的提醒。

## 范围

### 包含

- 扩展 Day 模块的日报数据结构，新增昨日改进提醒字段。
- 在构建每日日报数据时，读取昨天的 `daily` 总结记录。
- 在 `DailyReportModal` 中按需展示“昨日改进提醒”区块。
- 补充 composable 层测试，覆盖有内容、无内容和查询失败场景。

### 不包含

- 不修改 Summary 表结构。
- 不新增新的数据库表或 RPC。
- 不改变日总结表单字段。
- 不改变日报弹窗的出现频率和 `daily_report_log` 记录逻辑。
- 不做自动跳转 Summary 或总结编辑入口。

## 数据来源

昨日改进提醒来自 `summary` 表中昨天日期对应的 `daily` 总结：

- 查询入口：`db.summary.getByDateKind(yesterdayDate, 'daily')`
- 日期来源：`useDailyReport` 内计算出的昨天日期
- 字段来源：`summary.content.improve`

`getByDateKind` 已经基于 `kind + period_start/period_end` 查询，因此可以复用现有总结数据访问能力，不新增服务方法。

## 数据流

1. 用户进入今天的 Day 页面。
2. `useDailyReport.openIfNeeded()` 检查今天是否已经展示过日报。
3. 未展示过时调用 `buildStats()` 构建日报数据。
4. `buildStats()` 并行读取：
   - 昨日任务列表
   - 今日任务列表
   - 昨日 `daily` 总结记录
5. 从昨日总结中提取 `content.improve`，去除首尾空白后写入 `reportStats.yesterdayImprove`。
6. `DailyReportModal` 收到 `stats` 后，如果 `yesterdayImprove` 非空，则展示“昨日改进提醒”区块。

## UI 行为

`DailyReportModal` 保留现有结构：

- 标题区：每日日报 / 昨天复盘 · 今天计划
- 四格统计：昨天已完成、昨天未完成、今日任务数、顺延到今天
- 操作按钮：仅确认、将未完成任务顺延至今天

新增区块放在四格统计下方、操作按钮上方：

- 标题：`昨日改进提醒`
- 内容：展示昨天日总结中的“改进之处”
- 文本保留换行，支持多条改进事项
- 使用现有 Tailwind token，保持弹窗安静、清晰

当 `yesterdayImprove` 为空时不渲染该区块，避免出现无意义空态。

## 错误处理

- 昨天没有日总结：`yesterdayImprove` 为空，弹窗照常显示。
- 昨天日总结存在但 `improve` 为空：不展示提醒区块。
- 查询昨日总结失败：捕获错误并 `console.warn`，不阻塞任务统计和弹窗显示。
- 任务统计查询失败：沿用现有 `openIfNeeded()` 的错误处理，打印错误并释放 loading。

## 测试计划

### 自动化测试

- `useDailyReport` 在昨日总结存在且 `content.improve` 非空时，将文本写入 `reportStats.yesterdayImprove`。
- `useDailyReport` 在昨日总结不存在时，`yesterdayImprove` 为空字符串。
- `useDailyReport` 在昨日总结查询失败时仍然打开日报弹窗，并保持 `yesterdayImprove` 为空字符串。
- 如补充组件测试，则验证 `DailyReportModal` 仅在 `yesterdayImprove` 非空时渲染“昨日改进提醒”。

### 验证命令

- `pnpm vue-tsc`
- `pnpm build`

根据项目环境说明，`pnpm` 命令需要在沙盒外提权执行。

## 验收标准

- 今天首次进入日视图弹出每日日报时，如果昨天日总结填写过“改进之处”，弹窗中能看到该内容。
- 如果昨天没有填写“改进之处”，弹窗视觉与现有版本保持一致。
- 顺延任务和仅确认按钮行为不变。
- 类型检查和生产构建通过。
