## 一、P0：基础可用性与数据安全

这些功能优先级最高，属于继续开发前应尽快补齐的底座。

- [x] **删除操作统一二次确认**
   - 任务删除、习惯删除、目标删除、分类删除、总结删除等危险操作统一确认。
   - 目标删除需明确提示会影响月计划和日任务。

- [x] **习惯打卡防重复**
   - 防止快速点击、网络重试导致同一天重复打卡。
   - 可先做前端 / 服务层幂等保护，后续再补数据库唯一约束。

- [x] **乐观更新失败提示**
   - 写操作失败时不只 `console.error`，需要给用户 toast 反馈。
   - 涉及任务完成、习惯打卡、目标日计划完成、批量操作等。

- [ ] **新增 / 修改 / 删除统一节流或防重复提交**
   - 抽象统一的写操作锁或节流封装。
   - 覆盖创建任务、编辑任务、删除、打卡、批量创建 / 修改 / 删除。

- [ ] **关键类型与数据模型对齐**
   - 补齐 `Goal`、`Task`、`GoalDay` 等模型字段。
   - 收窄状态类型，减少运行时字段错误。
   - 逐步减少数据层 `as any`。

## 二、P1：时序与每日执行闭环

这些功能直接影响每天是否愿意继续使用 Rhythm。

1. **番茄钟中断路径**
   - 增加暂停、继续、放弃、退出确认。
   - 关闭弹窗时明确是否继续计时或放弃本次。

2. **今日收束面板**
   - 集中展示今天未完成任务。
   - 支持快速处理：稍后、明天、放弃。

3. **临时任务快速插入**
   - 只填写标题和预计时长即可创建。
   - 默认插入当前时间之后的合适空档。
   - 没有空档时提示用户重新选择时间，不新增“待安排”状态。

4. **执行中断轻记录**
   - 任务结束时支持轻量标记：完成、延期、被打断、取消。
   - 为后续复盘、AI 分析、时间估算提供数据。

5. **未完成任务顺延增强**
   - 支持快速顺延到稍后或明天。
   - 对缺少 `start_time` 的任务做安全处理。

6. **短任务卡片展示优化**
   - 当任务时长过短时，将描述放到标题后或以更紧凑形式展示。
   - 避免短任务卡片内容被截断后不可读。

7. **侧边栏点击滚动修复**
   - 点击侧边栏任务应稳定滚动到时间轴对应位置。
   - 需要覆盖普通任务、习惯任务、所向日计划、顺延项。

8. **时间指示线错位修复**
   - 打开控制台、窗口尺寸变化、滚动容器高度变化时，当前时间线不应错位。

9. **其他模块内容拖动加入时间轴**
   - 从侧边栏或其他模块拖动任务 / 目标 / 习惯到时间轴安排时间。
   - 需要先明确哪些来源可拖、拖入后写入哪张表。

10. **所向任务在时间轴中的编辑体验**
    - 当任务来源是所向目标时，编辑入口需要明确是编辑执行项，还是跳回所向结构。
    - 避免用户误以为所有时间轴任务都可以用同一个任务弹窗编辑。

11. **时间轴拖拽编排复核**
    - 旧清单未勾选，但现有功能文档已描述拖拽移动、缩放、5 分钟吸附。
    - 需要实际验证：桌面端是否稳定、拖拽后数据是否正确保存。

12. **时序页任务排序 / 编排**
    - 排序能力只聚焦 Day 时序页面。
    - 不做 Direction 目标分类排序、组内排序、月计划排序或日计划排序。
    - 已确定不是纯视觉排序，而是通过拖拽改变任务时间。
    - 支持时间轴内拖动任务改开始时间。
    - 支持从侧边栏拖动任务加入 / 调整到时间轴。

## 三、P2：复盘与总结闭环

这些功能用于降低复盘成本，让 Summary 不只是手写文本区。

1. **日报与日总结打通**
   - 日报确认后提供进入日总结的入口。
   - 日总结自动带入完成项、未完成项、延期项、习惯打卡。

2. **日总结自动草稿**
   - 根据当天执行事实生成初始内容。
   - 用户只需要修改和确认。

3. **周总结 / 月总结数据预填**
   - 自动聚合完成率、延期项、习惯坚持情况、目标推进。
   - 先生成页面内容，再考虑导出。

4. **未写总结提示**
   - Summary 侧边栏或日期视图标记未写日总结的日期。
   - 避免总结入口完全依赖用户记忆。

5. **周报 / 月报页面**
   - 展示任务完成、习惯坚持、目标推进、延期情况。
   - 后续可扩展为 Markdown / PDF 导出。

## 四、P3：数据反馈与可视化

这些功能用于增强长期使用动力。

1. **统计面板**
   - 任务完成率。
   - 每日专注时长。
   - 连续打卡。
   - 目标推进情况。
   - 入口确定为独立 Dashboard 页面。

2. **完成率趋势**
   - 展示近 7 天、近 30 天任务完成率变化。

3. **每日专注时长**
   - 基于番茄钟或任务实际开始 / 结束时间统计。

4. **一周任务热力图**
   - 类似 GitHub Contribution，用于展示执行稳定性。

5. **习惯长期反馈**
   - 连续打卡。
   - 周期完成率。
   - 月度完成率。
   - 习惯趋势对比。

6. **目标推进趋势**
   - 展示目标日计划完成情况。
   - 增强所向模块的长期反馈价值。

7. **月视图增强**
   - 每日完成率热力图。
   - 每天任务、目标、习惯的聚合指标。
   - 点击日期前可预览当天概要。

8. **年视图增强**
   - 月度节奏对比。
   - 年度习惯热力。
   - 年度任务完成趋势。
   - 年度目标推进概览。

9. **不同分类耗时占比**
   - 需要先统一任务、目标、习惯的分类口径。

10. **月度报告导出**
    - 在月报页面稳定后，再考虑 Markdown / PDF 导出。

## 五、P4：AI 与大模型能力

AI 能力应服务于“执行复盘闭环”，不要做成独立聊天入口。

1. **接入大模型**
   - 需要确定模型供应商、调用方式、密钥保存方式、是否走后端代理。

2. **AI 周总结 / 月总结草稿**
   - 基于任务完成率、延期项、习惯记录、目标推进、日总结生成草稿。
   - 已确定为 AI 第一批业务能力。

3. **AI 明日计划建议**
   - 基于未完成任务、明日空档、习惯、目标日计划生成建议安排。
   - 推荐“AI 给建议，用户确认后写入时间轴”。
   - 排在 AI 总结草稿之后。

4. **AI 拖延原因分析**
   - 基于延期、打断、取消等结构化原因分析拖延模式。
   - 需要先完成执行中断轻记录。

5. **AI 历史完成时间估算**
   - 根据历史实际开始 / 结束时间、延期、打断记录估算任务耗时。
   - 需要稳定记录真实执行数据。

## 六、P5：所向与组织能力

这些功能主要提升目标管理和长期编排效率。

1. **月任务完成后的展示策略**
   - 当一个月的目标日任务全部完成后，可以折叠、隐藏或标记完成。
   - 需要避免完成后突然消失造成困惑。

2. **目标今日快捷入口**
   - 直接查看今天来自所向的目标日计划。
   - 减少进入 Direction 后三级下钻的成本。

## 七、P6：主题、产品化与工程健康

这些功能适合在核心闭环稳定后推进。

1. **主题系统：亮色 / 暗色**
   - 与 Tailwind 语义 token、`src/assets/tw-theme.css` 保持一致。
   - 覆盖弹窗、导航、卡片、时间轴、热力图、表单、toast。

2. **Day 页面缓存**
   - 返回同一天时避免重复请求全部数据。
   - 可加入短 TTL 或手动刷新。

3. **Direction 缓存过期策略**
   - 避免多标签页或多设备修改后缓存长期不刷新。

4. **Store / Composable 边界继续收敛**
   - 避免 store 依赖 view 层 composable。
   - 减少 Direction 多 store 与共享 ref 的耦合。

5. **Supabase 类型生成**
   - 逐步减少 `as any`。
   - 让数据库字段变更能尽早在 TypeScript 阶段暴露。

6. **图标与 UI 性能优化**
   - lucide 图标按需导入或拆包。
   - 减少首屏加载体积。

7. **注册入口**
   - 当前如果未来要给别人使用，需要补自助注册。

8. **演示账号数据隔离或重置**
   - 避免访客之间互相污染数据。

9. **权限与安全策略增强**
    - 如果未来产品化，需要进一步检查 RLS、密钥暴露、数据导出安全。

## 九、已定决策

1. **不引入待安排状态**
   - 当前项目没有“待安排”状态。
   - 后续临时任务没有合适空档时，优先提示用户重新选择时间或直接顺延到稍后 / 明天。

2. **子任务暂不做**
   - 从当前待办范围移除。
   - 仅保留为远期可能方向，不进入近期实施计划。

3. **AI 第一批做周总结 / 月总结草稿**
   - 先让 AI 服务复盘闭环。
   - AI 明日计划建议后置。

4. **所向任务在 Day 里轻量编辑**
   - 当任务来源是所向目标时，Day 页面可以打开轻量编辑弹窗。
   - 轻量编辑只修改当天执行项，不修改上级目标或整个月计划。

5. **月任务全部完成后保留显示并标记完成**
   - 完成后的月任务不默认隐藏。
   - 通过完成状态降低视觉干扰，避免用户误以为数据消失。

6. **主题系统后置**
   - 亮色 / 暗色模式等核心闭环完成后再做。
   - 当前优先级低于执行、复盘、统计和 AI 总结草稿。

7. **大模型接入方式暂定**
   - 当前还未学习 / 确定大模型接入方案。
   - 后续项目可能整体迁移到 Java 后端，因此暂不选择前端直连或 Edge Function。
   - 等后端路线确定后，再决定大模型调用与密钥管理方式。

8. **协作功能不考虑**
   - 当前项目按个人自用效率系统推进。
   - 多人共享、团队任务、评论、@提醒、操作日志不进入路线图。

9. **统计面板使用独立 Dashboard 页面**
   - 不放进 Summary。
   - 不仅依赖 Month / Year 增强。
   - Month / Year 后续仍可展示局部数据反馈，但 Dashboard 作为总览入口。

10. **排序只聚焦时序页面**
    - 不做 Direction 分类排序、组内目标排序、月计划排序、日计划排序。
    - 后续排序 / 拖拽编排优先服务 Day 页面执行体验。
    - 已确定采用时间排序 / 拖拽编排：拖动后改写任务开始时间。
    - 支持侧边栏拖到时间轴，本质仍是改写任务时间。

11. **移动端单独做 React Native 项目**
    - 当前 Vue Web 项目不再继续补移动端适配。
    - 移动端体验后续作为独立 React Native 项目规划。
    - Web 项目待办中删除移动端 navbar、移动端弹窗、多端手机适配等事项。
